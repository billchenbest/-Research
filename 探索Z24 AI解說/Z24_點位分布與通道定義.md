## 一、EMS加速計點位分布

### 1.1 加速計位置總覽（根據 Fig. 37）

根據官方文件 **Fig. 37: Location and orientation of accelerometers**，Z24橋樑上共安裝了 **17個加速計**，分布如下：

- **通道 10**：最左側內側車道
- **通道 09**：中間區域內側車道
- **通道 08**：中間區域外緣

- **通道 12**：最右側外緣

- **通道 13**：中間區域外緣

#### 1.1.2 橋墩加速計（Cross View）

## 三、檔案命名規則與通道對應

### 3.1 EMS資料檔案命名規則

根據 **Appendix I: Description of Environmental Monitoring System Distributed Data**：

#### 3.1.1 檔案名稱前綴格式：`WWDHH`

| 代碼 | 說明 | 範圍/值 |
|------|------|---------|
| `WW` | 運行週數 | 01開始的兩位數 |
| `D` | 星期幾 | A=星期一, B=星期二, ..., G=星期日 |
| `HH` | 小時 | 00-23（24小時制） |

#### 3.1.2 加速度時間歷程檔案（`.AAA`）

**格式：** `WWDHH` + **通道號碼（兩位數）** + `.AAA`

**範例：**
- `13A2305.AAA` = 第13週，星期一，23時，**通道05**
- `01B1201.AAA` = 第1週，星期二，12時，**通道01**（橋墩下部）

#### 3.1.3 環境資料檔案（`.ENV`）

**格式：** `WWDHH` + `PRE`/`POS` + `.ENV`

- `PRE`：加速度測量**之前**的環境資料
- `POS`：加速度測量**之後**的環境資料

**範例：**
- `13A23PRE.ENV` = 第13週，星期一，23時，測量前環境資料
- `13A23POS.ENV` = 第13週，星期一，23時，測量後環境資料

### 3.2 資料分組

同一小時的所有測量資料（環境資料 + 加速度資料）會打包在同一個 **PkZip** 檔案中。

## 五、PDT測試通道定義

### 5.1 PDT測試資料結構

根據 **Appendix K: Progressive Damage Tests Distributed Data**：

**資料夾結構：**
```
pdt_01-08/
├── 01/
│   ├── avt/          # 環境振動測試
│   │   ├── 01setup01.mat
│   │   ├── 01setup02.mat
│   │   └── ...
│   └── fvt/          # 強迫振動測試
│       ├── 01setup01.mat
│       ├── 01setup02.mat
│       └── ...
```

### 5.2 `.mat` 檔案中的通道資訊

根據 `readme.txt` 說明：

**包含的變數：**
- `data`：時間資料矩陣
  - 每**列**（row）對應一個時間點
  - 每**行**（column）對應一個測量通道
- `labelshulp`：通道資訊標籤
  - 包含每個通道的測量位置資訊

### 5.3 PDT測試通道命名範例

從實際檔案分析中發現的通道命名模式：

**常見通道名稱：**
- `DP2V`：可能為激振輸入通道（建議作為參考通道）
- `R1V`, `R2V`, `R3V`：參考通道（可能為參考點）
- `139V`, `239V`, `343V`：測量點通道（數字可能對應測量網格編號）
- `139T`, `239T`：橫向（Transverse）測量通道

**命名規則推測：**
- 數字部分（如 139, 239）可能對應測量網格的行號
- 字母部分（V, T, L）對應方向：
  - **V**：垂直（Vertical）
  - **T**：橫向（Transverse）
  - **L**：縱向（Longitudinal）

> **注意**：完整的PDT測量網格定義應參考 **Appendix K** 中的 "Progressive Damage Tests Measurement Grid"（Page 120），但該頁面未包含在提供的圖片中。

## 六、座標系統定義

### 6.1 局部座標系統（Local Coordinate System）

根據 **Fig. 37**，Z24橋樑使用以下局部座標系統：

| 軸向 | 符號 | 方向說明 |
|------|------|---------|
| **垂直** | **V** | 向上（Vertical） |
| **橫向** | **T** | 水平向左（Transverse） |
| **縱向** | **L** | 水平向右（Longitudinal） |

### 6.2 通道名稱中的方向代碼

在通道名稱中，方向代碼通常出現在通道名稱的末尾：

- **`139V`**：測量點139，垂直方向
- **`239T`**：測量點239，橫向
- **`343L`**：測量點343，縱向

### 6.3 地理座標系統

- **North**：指向右上（Zurich方向）
- **South**：指向左下（Bern方向）
- **Utzenstorf**：橋樑左側（西側）
- **Koppigen**：橋樑右側（東側）

## 七、實際應用範例

### 7.1 從EMS檔案名稱識別通道位置

**範例1：** `13A2305.AAA`
- 第13週，星期一，23時
- **通道05** = Zurich方向，最左側外緣
- 方向：V/T/L（需查看實際資料確定）

**範例2：** `01B1201.AAA`
- 第1週，星期二，12時
- **通道01** = 橋墩下部（靠近基礎）
- 方向：V/T/L

### 7.2 從PDT檔案識別通道

**範例：** `pdt_01-08/01/fvt/01setup01.mat`

```python
import scipy.io as sio

# 載入資料
mat_data = sio.loadmat('pdt_01-08/01/fvt/01setup01.mat')
data = mat_data['data']  # 時間資料矩陣
labelshulp = mat_data['labelshulp']  # 通道資訊

# 查看通道標籤
print(labelshulp)  # 會顯示如：['DP2V', 'R1V', 'R2V', '139V', '239V', ...]

# 通道對應：
# 通道0 = DP2V（可能為激振輸入）
# 通道1 = R1V（參考通道1）
# 通道2 = R2V（參考通道2）
# 通道3 = 139V（測量點139，垂直方向）
# ...
```

### 7.3 通道選擇建議

**對於FRF計算：**
1. **輸入通道（Input）**：建議使用 `DP2V`（如果存在）
2. **參考通道（Reference）**：可使用 `R1V`, `R2V`, `R3V` 或 `DP2V`
3. **輸出通道（Output）**：選擇感興趣的測量點（如 `139V`, `239V` 等）

**對於轉角/曲率計算：**
1. 需要選擇**同一條梁線上的空間點位**（按順序）
2. 例如：`139V`, `140V`, `141V`, `142V`, `143V`（假設這些點在同一條線上）
3. 需要知道**相鄰點位距離 dx（公尺）**
